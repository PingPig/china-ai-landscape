# 热度指数计算方法论

## 数据来源（全部合法公开）

| 数据源 | 采集方式 | 更新频率 | 覆盖范围 |
|--------|---------|---------|---------|
| GitHub Stars | GitHub REST API（公开） | 实时快照 | 开源项目 |
| GitHub Forks | GitHub REST API | 实时快照 | 开源项目 |
| 融资金额 | 公开新闻/IT桔子/天眼查公开页 | 事件驱动 | 所有公司 |
| 产品数量 | 官网公开信息 | 季度更新 | 所有项目 |
| 重大事件 | 公开新闻报道 | 事件驱动 | 所有项目 |
| 上市市值 | 公开股票数据 | 季度快照 | 上市公司 |

## 热度指数计算（0-100分）

### 开源项目（有GitHub数据）

```
热度 = GitHub影响力(40%) + 融资规模(25%) + 产品广度(15%) + 事件热度(20%)

GitHub影响力 = normalize(log10(stars + forks * 2), 0, 100)
  - DeepSeek 398K stars → ~100
  - Dify 133K → ~88
  - Qwen 88K(合计) → ~83
  - MiniCPM 8.7K → ~50

融资规模 = {
  已上市且市值>100亿: 100
  已上市: 80
  E轮+/超100亿融资: 90
  D轮/超50亿: 75
  C轮/超20亿: 60
  B轮/超10亿: 50
  A轮/超5亿: 40
  天使/种子: 20
  未融资/自筹: 10
}

产品广度 = min(products.length * 15, 100)

事件热度 = 该季度相关重大事件数 * 30（上限100）
  - "high"影响力事件: 40分
  - "medium"影响力事件: 20分
```

### 闭源/非开源项目

```
热度 = 融资规模(30%) + 产品广度(20%) + 事件热度(25%) + 公开指标(25%)

公开指标 = {
  DAU > 1000万: 100
  DAU > 100万: 80
  DAU > 10万: 60
  有公开用户数据: 40
  无公开数据: 20
}
```

### 季度变化逻辑

- 基础热度按上述公式计算（反映项目实力）
- 季度事件造成波动（发布新模型+15~30，获重大融资+10~20，负面事件-10~20）
- 早期季度（项目未成立前）热度为0
- 成立首季度给予基础热度的30%，逐季增长

## 数据标注

所有数据在项目JSON中标注来源：
- `githubStars`: GitHub API实时获取，标注采集日期
- `funding`: 标注信息来源（如"36kr报道 2025.03"）
- `heatByQuarter`: 标注"按方法论v1.0计算"

## 免责声明

- 热度指数为综合评估指标，不代表商业价值或技术水平排名
- GitHub Stars仅反映开源社区关注度，不等同于技术实力
- 融资数据来源于公开报道，可能与实际金额有出入
